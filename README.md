# LMS-система — учебный проект на Django и DRF

## Описание

Это учебный проект системы управления обучением (LMS), разрабатываемый поэтапно. На каждом этапе реализуются новые
компоненты: от базовой настройки окружения до построения полноценного REST API для работы с курсами, уроками,
пользователями и платежами. К завершению проекта будет создан прототип LMS-платформы, пригодный для портфолио и
дальнейшего развития.

## Цели проекта

- Освоить разработку REST API на Django и Django REST Framework.
- Отработать практику работы с моделями, сериализаторами, вьюсетами и фильтрами.
- Научиться структурировать проект по приложениям.
- Реализовать полноценную CRUD-логику для основных сущностей.
- Получить опыт тестирования API с помощью Postman.

## Текущий статус

**Этап разработки:** Реализованы основные модели, API, сериализация и фильтрация.

## Этап 1: Базовая настройка и реализация основных сущностей

### Реализовано

- Создан новый Django-проект, подключен Django REST Framework (DRF).
- Реализовано приложение `users` с моделью пользователя:
    - Авторизация по email.

    - Реализована связь «один ко многим» между курсом и уроками.
- Описаны CRUD-операции:
    - Для курса — через Viewsets.
    - Для урока — через Generic-классы.
- Реализованы простейшие сериализаторы для всех моделей.
- Проверена работа всех эндпоинтов с помощью Postman.

## Этап 2: Расширение функционала и оптимизация API

### Реализовано

- В сериализатор курса добавлено поле количества уроков через `SerializerMethodField`.
- В приложение `users` добавлена модель **Платежи**:
    - Пользователь (ForeignKey).
    - Дата оплаты.
    - Оплаченный курс или урок (ForeignKey).
    - Сумма оплаты.
    - Способ оплаты (наличные/перевод).

- Настроена фильтрация для эндпоинта списка платежей:
    - Сортировка по дате оплаты.
    - Фильтрация по курсу/уроку.
    - Фильтрация по способу оплаты.
- Дополнительное задание: в сериализатор пользователя добавлено поле истории платежей пользователя.

## Этап 3: Управление доступом и авторизация

### Реализовано

- **Реализована CRUD-логика для пользователей**, включая регистрацию новых пользователей.
- **Настроена JWT-авторизация** в проекте. Все эндпоинты, кроме регистрации и авторизации, закрыты для неавторизованных
  пользователей.
- **Создана группа «Модераторы»** с ограниченными правами: модераторы могут просматривать и редактировать любые курсы и
  уроки, но не могут их создавать или удалять.
- **Реализован кастомный permission-класс** для проверки принадлежности пользователя к группе модераторов (используется
  метод `has_permission()` и проверка через `request.user.groups.filter(name='Moderators').exists()`).
- **Группы пользователей** можно назначать через админ-панель. Для автоматизации создана фикстура или кастомная команда
  для заполнения групп в базе данных.
- **Разграничены права доступа**:
    - Для ViewSet-права определяются динамически через метод `get_permissions()` для разных action (например, создание,
      просмотр, редактирование).
    - Для Generic-классов права задаются через атрибут `permission_classes`.
- **Реализована объектная авторизация**: пользователи, не являющиеся модераторами, могут просматривать, редактировать и
  удалять только свои курсы и уроки. Для этого в моделях добавлено поле владельца, а при создании объекта используется
  метод `perform_create()` для привязки к текущему пользователю.
- **Привязка объектов к владельцу** реализована во всех необходимых контроллерах (например, при создании урока или
  курса).
- **Объединение и динамическое управление правами** с помощью логических операторов (`&`, `|`, `~`) в
  permission-классах.

### Примеры настройки прав

```python
# Пример для ViewSet
def get_permissions(self):
    if self.action == 'create':
        self.permission_classes = [IsAuthenticated]
    elif self.action in ['list', 'retrieve']:
        self.permission_classes = [IsAuthenticated]
    elif self.action in ['update', 'partial_update']:
        self.permission_classes = [IsAuthenticated, IsModerator | IsOwner]
    elif self.action == 'destroy':
        self.permission_classes = [IsAuthenticated, IsOwner]
    return [permission() for permission in self.permission_classes]


# Пример для GenericAPIView
class LessonListAPIView(generics.ListAPIView):
    ...
    permission_classes = [IsAuthenticated]


class LessonUpdateAPIView(generics.UpdateAPIView):
    ...
    permission_classes = [IsAuthenticated, IsModerator | IsOwner]
```

### Технические детали

- **JWT-авторизация** реализована с помощью стандартных библиотек DRF.
- **Права доступа** проверяются как на уровне списка/просмотра, так и на уровне изменения/удаления объектов.
- **Все операции** по созданию, редактированию и удалению объектов защищены согласно ролям и владельцам.

### Тестирование

- Проведено тестирование через Postman для всех сценариев доступа.

## Этап 4: Валидаторы, подписки, пагинация и тесты

### Реализовано

- **Валидатор ссылок на видео**: создан отдельный файл `validators.py` с валидатором, который разрешает только ссылки на
  *youtube.com*. Валидатор интегрирован в сериализатор урока. При попытке добавить стороннюю ссылку возвращается ошибка.
- **Модель подписки на курс**: реализована модель с полями «пользователь» (*FK* на пользователя) и «курс» (*FK* на
  курс). Добавлен эндпоинт для управления подпиской (добавление/удаление) через *APIView* и метод *POST*. В сериализатор
  курса добавлено поле, показывающее, подписан ли текущий пользователь на обновления курса.
- **Пагинация для курсов и уроков**: создан файл `paginators.py` с классами пагинации (например, на основе
  *PageNumberPagination*), заданы параметры: *page_size*, *page_size_query_param*, *max_page_size*. Пагинатор подключён
  к соответствующим контроллерам через параметр `pagination_class`.
- **Тестирование**: написаны тесты для *CRUD* уроков и функционала подписки. В тестах используется метод `setUp` для
  подготовки тестовых данных, реализованы проверки для пользователей с разными правами доступа. Для аутентификации
  применяется `self.client.force_authenticate()`.

### Технические детали

- Валидатор ссылок реализован как функция или класс и подключён к нужному полю сериализатора.
- Эндпоинт подписки возвращает сообщение о результате действия (подписка добавлена/удалена).
- Пагинация работает для всех списковых эндпоинтов курсов и уроков, параметры можно передавать через запрос.
- Тесты покрывают основные сценарии работы с уроками и подписками, включая проверку прав доступа.

### Тестирование

- Проведено тестирование новых функций через *Postman* и автотесты.
- Проверена корректность работы валидатора, подписки, пагинации и разграничения прав.
