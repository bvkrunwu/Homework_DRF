# LMS-система — учебный проект на Django и DRF

## Описание

Это учебный проект системы управления обучением (LMS), разрабатываемый поэтапно. На каждом этапе реализуются новые
компоненты: от базовой настройки окружения до построения полноценного REST API для работы с курсами, уроками,
пользователями и платежами. К завершению проекта будет создан прототип LMS-платформы, пригодный для портфолио и
дальнейшего развития.

## Цели проекта

- Освоить разработку REST API на Django и Django REST Framework.
- Отработать практику работы с моделями, сериализаторами, вьюсетами и фильтрами.
- Научиться структурировать проект по приложениям.
- Реализовать полноценную CRUD-логику для основных сущностей.
- Получить опыт тестирования API с помощью Postman.

## Текущий статус

**Этап разработки:** Реализованы основные модели, API, сериализация и фильтрация.

---

## Этап 1: Базовая настройка и реализация основных сущностей

### Реализовано

- Создан новый Django-проект, подключен Django REST Framework (DRF).
- Реализовано приложение `users` с моделью пользователя:
    - Авторизация по email.
    - Дополнительные поля: телефон, город, аватарка.
- Создано приложение `lms` (или `materials`) с моделями:
    - **Курс**: название, превью, описание.
    - **Урок**: название, описание, превью, ссылка на видео.
    - Реализована связь «один ко многим» между курсом и уроками.
- Описаны CRUD-операции:
    - Для курса — через Viewsets.
    - Для урока — через Generic-классы.
- Реализованы простейшие сериализаторы для всех моделей.
- Проверена работа всех эндпоинтов с помощью Postman.

---

## Этап 2: Расширение функционала и оптимизация API

### Реализовано

- В сериализатор курса добавлено поле количества уроков через `SerializerMethodField`.
- В приложение `users` добавлена модель **Платежи**:
    - Пользователь (ForeignKey).
    - Дата оплаты.
    - Оплаченный курс или урок (ForeignKey).
    - Сумма оплаты.
    - Способ оплаты (наличные/перевод).
- В сериализатор курса реализовано поле вывода всех уроков курса (вложенная сериализация).
- Один сериализатор курса теперь возвращает и количество уроков, и список уроков.
- Настроена фильтрация для эндпоинта списка платежей:
    - Сортировка по дате оплаты.
    - Фильтрация по курсу/уроку.
    - Фильтрация по способу оплаты.
- Дополнительное задание: в сериализатор пользователя добавлено поле истории платежей пользователя.

## Тестирование API

Все эндпоинты протестированы с помощью Postman. Примеры запросов:

- **GET** `/api/courses/` — список курсов
- **POST** `/api/lessons/` — создание урока
- **GET** `/api/payments/?course=1&ordering=-date` — фильтрация платежей по курсу и сортировка по дате
